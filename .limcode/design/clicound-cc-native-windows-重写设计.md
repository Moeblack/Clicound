# Clicound C/C++ 原生 Windows 版设计

## 一、总体思路

用纯 C/纯 Win32 API 写一个单 EXE、零外部依赖的系统托盘常驻程序。不用任何 GUI 框架（不用 Qt、不用 MFC、不用 Electron），只用 Windows 自带的东西。这样编译出来的 EXE 大约 100~300KB，启动瞬间完成，内存占用几 MB，和系统完全融为一体。

### 为什么选 C 而不是 C++

这个项目的逻辑非常简单（挂钩子、播声音、画对话框），C 完全够用，而且 Win32 API 本身就是 C 接口。唯一需要 C++ 的地方是 XAudio2（COM 接口），但可以用 C 的 COM 调用方式绕过，或者在这一小块用 C++ 编译。实际做法：**用 C++ 编译，但写法以 C 风格为主**，只在 XAudio2 初始化处用 C++ 语法。

## 三、核心技术选型

### 3.1 全局鼠标监听：WH_MOUSE_LL

用 `SetWindowsHookEx(WH_MOUSE_LL, callback, hInstance, 0)` 安装低级鼠标钩子。

关键要点：
- **WH_MOUSE_LL 不需要注入 DLL**，回调直接跑在安装钩子的进程里，这是它和 WH_MOUSE 的本质区别。
- 回调里必须快速返回（Windows 限制超时 ~300ms，Win10 1709+ 最大 1000ms），所以播放操作必须是异步非阻塞的。
- 回调里拿 `wParam` 判断事件类型：`WM_LBUTTONDOWN`、`WM_MBUTTONDOWN`、`WM_RBUTTONDOWN`、`WM_MOUSEWHEEL`。
- 主线程需要跑 `GetMessage/DispatchMessage` 消息循环来驱动钩子。

**滚轮事件的特殊性**：

`WM_MOUSEWHEEL` 和按键点击有本质区别。按键是离散的（按一次响一声），滚轮是连续/高频的。具体来说：

- 传统鼠标滚轮：每滚一格发送一次事件，`MSLLHOOKSTRUCT.mouseData` 高16位为 ±120（`WHEEL_DELTA`）。滚动速度适中时大约每秒 5~15 个事件。
- 精密触摸板（Windows Precision Touchpad）：双指滑动会发送更小增量（如30、60），频率极高，可达每秒 50~100 个事件。
- 水平滚轮（`WM_MOUSEHWHEEL`）：同理，触摸板双指水平滑动触发。

因此滚轮音效需要专门的**节流策略**和不同于点击的**音效设计**，详见下文 3.6 节。

### 3.2 音频播放：XAudio2

不用 `PlaySound`（阻塞、延迟高），不用 `waveOut`（过时、API 复杂），用 **XAudio2**：

- Windows 10/11 内置，零额外依赖
- 支持预加载 PCM buffer，播放延迟极低（<5ms）
- 支持多声音同时播放（多 SourceVoice）
- 可独立设置每个声音的音量

初始化流程：
1. `CoInitializeEx` → `XAudio2Create` → `CreateMasteringVoice`
2. 预读 WAV 文件到内存 buffer（左键、中键、右键、滚轮向上、滚轮向下，共 5 个）
3. 每次点击时：取一个空闲 SourceVoice → SubmitSourceBuffer → Start
4. 用 voice pool（比如 8 个预创建的 SourceVoice）避免每次点击都创建/销毁

### 3.6 滚轮音效设计

滚轮是整个项目中最需要精心设计的部分。做得好，触摸板双指滑动会有非常舒适的"棘轮感"；做得差，会变成一团嘈杂的噪音。

#### 音效特征设计

滚轮音效和点击音效的设计目标完全不同：

| 维度 | 点击音效 | 滚轮音效 |
|------|---------|---------|
| 目的 | 确认"操作已执行" | 传达"正在滚动"的连续感 |
| 时长 | 10~20ms | 5~8ms（必须更短） |
| 音量 | 正常 | 比点击低 30~50%（否则连续播放很吵） |
| 频率 | 1200~1800Hz | 2500~4000Hz（更高更细，像棘轮的"嘀嘀"） |
| 衰减 | 中等衰减 | 极快衰减（指数衰减系数更大） |
| 听感目标 | 机械键盘的"咔" | 苹果 Digital Crown 的"嗒嗒嗒" |

**为什么要更高频率、更短时长**：滚轮事件密集时（每秒几十次），低频长音会糊成一片嗡嗡声。高频短促的"嘀"声即使快速连播也能保持颗粒感，每一声之间有清晰的间隙，像物理棘轮的齿轮咬合声。

**区分上下方向**：滚轮向上和向下使用略微不同的音调（比如向上 3200Hz、向下 2800Hz），提供方向感。音高差异不需要很大，只要能在潜意识层面感知到"声音变了"即可。这借鉴了苹果 watchOS Digital Crown 的设计——顺时针和逆时针旋转音调微妙不同。

#### 默认音效参数

```
scroll_up.wav:
  频率: 3200 Hz
  时长: 6ms
  音量系数: 0.15（比点击的 0.30~0.35 低很多）
  衰减系数: exp(-t * 600)（点击是 300，滚轮衰减快一倍）

scroll_down.wav:
  频率: 2800 Hz
  时长: 6ms
  音量系数: 0.15
  衰减系数: exp(-t * 600)
```

#### 滚轮节流策略

直接每个 `WM_MOUSEWHEEL` 事件都播放一次音效会出问题：

1. **精密触摸板的高频小增量**：双指快速滑动时每秒 50~100 个事件，每个只有 30 的 delta，如果全部播放会变成刺耳的蜂鸣。
2. **音效叠加爆音**：大量短音效同时播放会导致波形叠加，峰值超限。

节流方案：**基于 delta 累积 + 最小间隔双重门控**

```
全局状态：
  accumulated_delta = 0      // 累积的滚轮增量
  last_scroll_sound_time = 0 // 上次播放滚轮音效的时间戳

每次收到 WM_MOUSEWHEEL：
  delta = HIWORD(mouseData)  // 带符号，正=向上，负=向下
  accumulated_delta += delta

  // 门控条件1：累积量达到一个"格"（WHEEL_DELTA=120）
  // 门控条件2：距离上次播放至少 25ms（限制最大频率 ~40次/秒）
  if |accumulated_delta| >= 60 AND (now - last_scroll_sound_time) >= 25ms:
      if accumulated_delta > 0:
          play(scroll_up)
      else:
          play(scroll_down)
      accumulated_delta = 0
      last_scroll_sound_time = now
```

**为什么累积阈值是 60 而不是 120**：设为半个 WHEEL_DELTA 是折中。传统鼠标每格就是 120，直接满足条件每格响一声。精密触摸板的小增量（30、60）会累积两三次才响一声，避免过于密集。用户可以在设置中调整这个灵敏度。

**为什么最小间隔是 25ms**：对应最大 40 次/秒的播放频率。6ms 的音效在 25ms 间隔下有 19ms 的静默间隙，足以保持颗粒感。人耳对 25ms 间隔的连续短声感知为"快速滴答"而非"连续音"，这正是棘轮感的来源。

### 3.3 系统托盘：Shell_NotifyIcon

- 用 `Shell_NotifyIconW` + `NOTIFYICONDATAW` 添加托盘图标
- 托盘图标收到 `WM_LBUTTONUP` 打开设置对话框
- 托盘图标收到 `WM_RBUTTONUP` 弹出右键菜单
- 右键菜单项：设置、暂停/恢复、开机自启、退出

### 3.4 设置对话框：DialogBox + 资源模板

用 `.rc` 资源文件定义对话框模板，`DialogBoxParam` 弹出模态对话框。优点是系统原生渲染，自动跟随 Windows 主题（亮色/暗色在 Win11 上需要额外处理，但基本控件会跟随）。

### 3.5 配置持久化：JSON

继续用 config.json，但不引入第三方 JSON 库（避免依赖）。两个选择：
- **方案 A**：用 cJSON（单个 .c/.h 文件，MIT 协议，可以直接嵌入项目）
- **方案 B**：用简单的 INI 格式代替 JSON

推荐方案 A，cJSON 足够轻量，和 Python 版的配置文件格式保持一致。

## 四、UI 设计

### 4.1 系统托盘行为

这个工具的核心交互哲学是 **装好就忘**。用户不应该需要频繁打开 UI，所以主界面就是系统托盘图标，不应该有主窗口。

**托盘图标**：
- 一个小的扬声器/点击波纹图标
- 左键单击 → 打开设置对话框
- 右键单击 → 弹出菜单

**右键菜单项**：


#### 每个控件的设计理由

**音效文件区域**：
- 文本框显示当前路径，不可手动编辑（防止输错路径），点"浏览"弹出 `GetOpenFileName` 文件选择对话框，过滤 `.wav` 文件。
- 为什么不做拖拽？Win32 原生对话框做 drag-drop 要额外写 OLE 代码，投入产出比不高。
- 点击和滚轮分成两个分组框（GroupBox），视觉上区分两类不同性质的音效。

**启用复选框**：
- 四个 CheckBox（左键、中键、右键、滚轮），允许用户关闭不需要的音效。
- 滚轮单独一个开关而不是拆成"向上"和"向下"，因为很少有人只想听一个方向的声音。

**音量滑块**：
- 一个 Trackbar 控件 + 右侧百分比文字。
- 旁边放一个"试听"按钮，点击后播放一次左键音效，让用户不用真的去点其他地方就能听到效果。这个细节非常重要，否则用户调完音量不知道效果如何。
- 这个音量同时控制点击和滚轮。滚轮音效本身在生成时已经比点击低很多（0.15 vs 0.35），所以统一调节不会出现滚轮太吵的问题。

**滚轮灵敏度滑块**：
- 控制滚轮节流的累积阈值。最左端（低灵敏度）= 阈值 120（一整格才响一声），最右端（高灵敏度）= 阈值 30（几乎每个小增量都响）。
- 默认值放在中间偏低（阈值 60），适合大多数触摸板。传统鼠标用户可以调到最低，精密触摸板用户可以根据手感调高。

**触发时机**：
- 两个 RadioButton：按下时发声 vs 释放时发声。
- 大多数人习惯按下时有反馈（模拟物理按键的触底反馈），但有些人可能偏好释放时。默认选"按下时"。

**确定/取消**：
- 确定 = 保存配置 + 关闭对话框 + 立即生效
- 取消 = 不保存 + 关闭对话框

### 4.3 UX 细节

**首次启动体验**：
- 首次运行时弹出一个气泡通知："Clicound 已启动，在系统托盘中可以找到我"，附带托盘图标的指示。
- 不要弹出设置对话框，默认配置已经可以直接用。

**声音防抖**：
- 如果用户双击（两次点击间隔 <30ms），只播放一次音效，避免叠加产生刺耳的效果。

**优雅退出**：
- 退出前卸载钩子、释放 XAudio2 资源、删除托盘图标。
- 处理 `WM_QUERYENDSESSION`，在系统关机时也能优雅清理。

**开机自启**：
- 写入 `HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run`，值为 EXE 的完整路径。
- 勾选/取消时即时生效，不需要点"确定"。

**单实例保护**：
- 用 Named Mutex（`CreateMutex`）确保只有一个实例在运行。
- 如果已有实例在运行，第二次启动时把已有实例的设置对话框弹到前台，然后自己退出。

### 链接的系统库

- `user32.lib`：窗口、消息、钩子
- `shell32.lib`：托盘图标
- `comdlg32.lib`：文件对话框
- `advapi32.lib`：注册表（开机自启）

全部是 Windows SDK 自带，不需要任何第三方库（除了嵌入的 cJSON）。

### app.manifest

```xml
<assembly xmlns="urn:schemas-microsoft-com:asm.v1" manifestVersion="1.0">
  <dependency>
    <dependentAssembly>
      <assemblyIdentity type="win32" name="Microsoft.Windows.Common-Controls"
        version="6.0.0.0" processorArchitecture="*" publicKeyToken="6595b64144ccf1df"/>
    </dependentAssembly>
  </dependency>
  <application xmlns="urn:schemas-microsoft-com:asm.v3">
    <windowsSettings>
      <dpiAware xmlns="http://schemas.microsoft.com/SMI/2005/WindowsSettings">true/pm</dpiAware>
      <dpiAwareness xmlns="http://schemas.microsoft.com/SMI/2016/WindowsSettings">PerMonitorV2</dpiAwareness>
    </windowsSettings>
  </application>
</assembly>
```

这个 manifest 做两件事：
1. 启用 Common Controls v6，让对话框控件使用现代视觉样式（不会看起来像 Windows 2000）
2. 声明 DPI 感知，在高分屏上不会糊

## 六、与 Python 版的对比

| 维度 | Python 版 | C/C++ 原生版 |
|------|----------|-------------|
| EXE 大小 | 需要 Python 运行时 | ~100-300KB 单文件 |
| 启动速度 | ~1-2秒 | 瞬间 |
| 内存占用 | ~30-50MB | ~3-5MB |
| 音频延迟 | ~11ms (pygame) | <5ms (XAudio2) |
| 系统集成 | 终端窗口 | 系统托盘，无窗口 |
| 开机自启 | 需要额外配置 | 内置支持 |
| 依赖安装 | 需要 pip/uv | 零依赖 |
| 开发难度 | 低 | 中等 |

## 七、实现优先级

1. **P0 - 核心功能**：鼠标钩子（点击 + 滚轮 + 滚轮节流） + XAudio2 播放 + 消息循环 → 能跑起来能发声
2. **P1 - 系统集成**：托盘图标 + 右键菜单（暂停/退出）+ 单实例
3. **P2 - 设置 UI**：设置对话框 + 配置读写
4. **P3 - 完善体验**：开机自启、点击防抖、滚轮灵敏度滑块、试听按钮、触发时机选择
